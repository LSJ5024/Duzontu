두쫀투(두근두근 쫀득한 투자) - MVP 제품 요구사항 정의서 (PRD)
1. 프로젝트 개요 (Project Overview)
제품명: 두쫀투 (웹 기반 MVP)

제품 목표: 개인 투자자가 겪는 금융 정보의 과부하를 해소하기 위해, 방대한 어닝콜과 거시경제 지표를 AI가 대리 분석하여 웹 대시보드에 직관적으로 제공하고 맞춤형 포트폴리오 리밸런싱 전략을 제안한다.

개발 전략: 자체 AI 모델을 바닥부터 개발하는 막대한 비용과 시간을 절약하기 위해 OpenAI(GPT) 또는 Google Gemini의 상용 API를 활용하여 2주 내에 시장의 핵심 수요를 검증한다.

2. 대상 사용자 (Target Audience)
기업 실적 발표 원문이나 거시경제 뉴스를 직접 분석할 시간과 전문 지식이 부족한 개인 투자자.

시장 변동성(관세, 금리 등)에 대응해 자신의 주식 포트폴리오를 어떻게 조정해야 할지 객관적인 가이드가 필요한 투자자.

3. 핵심 기능 (Core Features - In Scope)
기능 1: 카테고리별 어닝콜 대리 분석 및 자동 포스팅 (시가총액 100위 큐레이션)
기능 요약: 미국 증시 시가총액 상위 100개 타깃 기업의 어닝콜 트랜스크립트를 AI가 분석하여 지정된 양식으로 웹 게시판에 자동 발행하며, 카테고리(섹터/테마)별 필터링 기능과 시가총액 기반 정렬을 제공한다.

사용자 스토리 1: "사용자는 메인 화면에 접속하여 최신 어닝콜 요약본들을 기업 시가총액 순서대로 한눈에 확인할 수 있다."
사용자 스토리 2: "사용자는 상단 필터에서 '의료 AI', '반도체' 등 특정 카테고리 태그를 클릭하여, 해당 섹터나 테마에 속한 100위권 내 기업들의 어닝콜 결과만 즉시 모아서 볼 수 있다."

작동 로직:
1. 시스템이 대상 유니버스(미국 증시 시가총액 상위 100개 종목 리스트)를 스케줄링하여 외부 금융 Data API에서 최신 어닝콜 원문을 수집한다.
2. 프롬프트 엔지니어링이 적용된 상용 LLM API(Gemini/GPT)로 원문을 전송한다. 이때 AI가 단순 줄글이 아닌 '성장 전망', '거시경제(관세/인플레이션) 영향', '리스크 요인' 요약과 함께, 기업이 속한 테마 카테고리(예: 반도체, 의료AI 등) 태그 메타데이터를 JSON 포맷으로 반환하도록 강제한다.
3. 데이터베이스(AWS RDS 등)에 반환된 JSON 분석 데이터를 저장할 때, 대상 기업의 '시가총액' 수치와 '카테고리(섹터)' 정보를 함께 적재(Mapping)한다.
4. 웹 프론트엔드 UI에서는 피드를 렌더링할 때 기본 옵션으로 '시가총액 높은 순'으로 정렬시켜 노출하고, 상단의 카테고리 버튼 클릭 시 해당 태그를 가진 포스트만 즉각 필터링하여 보여준다.

기능 2: Vision AI 기반 포트폴리오 스크린샷 분석 및 맞춤형 리밸런싱 제안
기능 요약: 직접적인 주식 계좌 연동이 어려운 MVP의 한계를 극복하기 위해, 사용자가 자신의 증권사 앱(보유 주식 화면) 스크린샷을 업로드하면 Vision AI가 이를 판독하고 사용자가 선택한 투자 성향에 맞춰 최적의 리밸런싱 전략을 제안한다.

사용자 스토리: "사용자는 자신의 증권 계좌 종목 화면을 캡처해 업로드한 뒤 '안정적인(배당성장 위주)' 옵션을 클릭한다. AI는 사진 속 종목들을 자동 인식하고, 최신 어닝콜 상황을 반영해 배당주 위주로 비중을 어떻게 조정해야 할지 맞춤형 투자 조언을 즉시 제공한다."

작동 로직:
1. 사용자가 웹 인터페이스에 자신의 보유 주식 현황이 담긴 스크린샷 이미지 1장을 업로드한다.
2. 사용자는 세 가지 투자 조언 성향 중 하나를 선택한다:
   - ① 공격적인 (개별 투자 위주)
   - ② 안정적인 (배당성장 위주)
   - ③ 미래 지향적인 (S&P 500, 나스닥 100 등 ETF 위주)
3. 프론트엔드는 업로드된 이미지와 선택된 성향 데이터를 백엔드로 넘기고, 백엔드는 이를 멀티모달(Vision)을 지원하는 상용 LLM API (예: GPT-4o 또는 Gemini 1.5 Pro)에 전송한다.
4. Vision AI가 스크린샷 내의 보유 종목명, 수익률, 비중 등을 OCR 판독하고, 이를 바탕으로 사용자의 성향에 맞는 리밸런싱 가이드(편입/편출/유지 비율 및 사유)를 구성해 웹 화면에 출력한다.

4. 제외 범위 (Out of Scope - 추후 개발)
iOS/Android 네이티브 모바일 앱 개발 (현재는 반응형 웹사이트로 통일).

자체 오픈소스 LLM 호스팅 및 파인튜닝(Fine-Tuning) 구축.

증권사 계좌 연동 자동 매매 기능 및 유료 결제/구독 시스템.

5. 기술 아키텍처 및 방향성 (Tech Stack & Direction)
프론트엔드 (Frontend 웹 개발):
- 제안 스택: Next.js (React 기반) + TypeScript + Vanilla CSS (또는 Tailwind CSS)
- 선정 이유: MVP를 빠르게(2주 이내) 검증하기 위해서는 풍부한 생태계와 재사용성이 뛰어난 React 기반의 Next.js가 가장 최적화된 방법입니다. 순수 Java(JSP 등)로 웹 프론트를 구성하는 것은 최신 트렌드나 개발 속도 측면에서 비효율적입니다. Next.js는 배포가 빠르고 AI API를 연동할 수 있는 내부 서버(API Route) 기능도 제공하여 풀스택 개발에 유리합니다.

백엔드 & 데이터베이스 (AWS 환경 전환):
- 기존: Supabase
- 대안 스택: AWS RDS (PostgreSQL/MySQL) + AWS EC2/Lambda
- AWS 활용 방법: 자체 서버를 두지 않고, 완전 관리형 클라우드 데이터베이스인 AWS RDS를 프로비저닝하여 사용합니다. 서버의 경우 Next.js 호스팅은 Vercel이나 AWS Amplify를 사용하고, AI 텍스트 처리가 무거워질 경우 Python(FastAPI) 백엔드 서버를 AWS EC2나 Lambda에 띄워 API로 통신하는 방법이 장기적 확장성 측면에서 우수합니다.

AI 및 데이터 처리: OpenAI API (GPT-4o) 또는 Google Gemini API

6. 어닝콜 및 거시경제 데이터 수집 파이프라인 (Data Pipeline)
효율적인 데이터 수집과 2주 내의 신속한 MVP 완성을 위해 강력한 봇 방어막 우회 등 불필요한 공수가 드는 **직접 크롤링을 완전히 배제하고, 신뢰성 높은 외부 전문 Data API를 활용**하는 전략으로 확정합니다.
- 데이터 소스: FMP(Financial Modeling Prep) 공식 API 또는 API 마켓플레이스(RapidAPI)의 비공식 래퍼(Wrapper) API (Seeking Alpha/Yahoo Finance 기반 등).
- 수집 포맷: 어닝콜 트랜스크립트(Transcript) 전문을 텍스트 또는 JSON 형태로 API 통신을 통해 즉각 반환받습니다.
- 비용 및 플랜 최적화 (1개월 MVP 전략):
  - 1순위 (무료/Freemium안): RapidAPI를 통해 월 무료 할당량(예: 500건 내외) 이내로 MVP를 테스트합니다.
  - 2순위 (유료/단기 해지안): 데이터 퀄리티나 안정성이 부족할 경우, 공식 벤더 프리미엄(월 $30~$50)을 결제하되 1달 단위 MVP 검증 직후 해지하여 개발 인건비와 소요 시간을 방어합니다.
- 기술적 이점: 타겟 사이트 UI 변경이나 캡챠(Cloudflare) 같은 봇 차단막을 뚫기 위한 코드 수정이 일절 필요 없으며, Next.js 백엔드 환경에서 곧바로 안전하게 API 호출만 수행하면 되므로 극강의 안정성을 지닙니다.

7. 상용 AI (Gemini, ChatGPT) 연동 상세 방법론
- 프롬프트 엔지니어링 및 JSON 강제 출력: AI에게 단순 질문이 아닌, 페르소나와 제약사항을 부여하는 System Prompt를 설계합니다. 특히 GPT의 `JSON Mode` 또는 Gemini의 `Structured Outputs` 기능을 사용하여, 응답을 단순 줄글이 아닌 `{ "growth": "...", "risk": "..." }` 형태의 정형화된 JSON 포맷으로 강제 반환받도록 처리합니다. 이를 통해 즉시 DB(AWS RDS)에 저장하고 프론트엔드로 뿌려줄 수 있습니다.
- 백엔드 프록시 아키텍처 구축: 보안(API 키 노출 방지)을 위해 웹 프론트엔드(브라우저)에서 직접 AI API를 호출하지 않습니다. 반드시 "유저 브라우저 → 우리 백엔드 서버(Next.js API 또는 FastAPI) → OpenAI/Gemini API" 순으로 호출하며, 우리 백엔드에서 사용자 요청을 로깅하고 제한(Rate limiting)을 둡니다.
- RAG 파이프라인: 위 6번 파이프라인에서 수집한 어닝콜 텍스트 원문을 프롬프트에 `[Context Document]`로 주입하여 "제공된 문서 내에서만 답변하라"는 지시를 내림으로써 환각을 차단합니다.

8. 제약 사항 및 리스크 관리 (Constraints & Risks)
할루시네이션(환각) 통제: LLM이 존재하지 않는 재무 지표를 지어내지 않도록, 외부 지식을 프롬프트에 주입하여 팩트 기반의 답변만 생성하게 하는 RAG 방식을 철저히 적용한다.

토큰 비용 최적화: 무분별한 API 과금을 막기 위해, 텍스트 요약 시 불필요한 서론/결론을 제외한 핵심 데이터만 추출 전송하고, 로컬 캐싱(Redis 등)을 통해 동일 분석 결과는 재요청하지 않도록 설계한다.
